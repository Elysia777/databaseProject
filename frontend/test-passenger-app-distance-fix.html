🔧 乘客端距离信息修复测试

🐛 问题发现

根本问题：
passenger-app.html 中的 confirmOrder() 函数没有包含距离、时长和费用信息！
这导致即使后端代码正确，数据库也无法保存这些信息。

🔍 问题分析

❌ 修复前的订单数据

const orderData = { passengerId: parseInt(passengerId), pickupAddress: startAddress, pickupLatitude: startPos.lat, pickupLongitude: startPos.lng, destinationAddress: endAddress, destinationLatitude: endPos.lat, destinationLongitude: endPos.lng, orderType: 'REAL_TIME' // ❌ 缺少距离、时长、费用信息 };

✅ 修复后的订单数据

// 获取路线信息 let estimatedDistance = 0; let estimatedDuration = 0; let estimatedFare = 0; if (window.routeInfo) { estimatedDistance = parseFloat((window.routeInfo.distance / 1000).toFixed(2)); estimatedDuration = Math.round(window.routeInfo.duration / 60); estimatedFare = parseFloat(document.getElementById('estimatedPrice').textContent.replace('¥ ', '')); } const orderData = { passengerId: parseInt(passengerId), pickupAddress: startAddress, pickupLatitude: startPos.lat, pickupLongitude: startPos.lng, destinationAddress: endAddress, destinationLatitude: endPos.lat, destinationLongitude: endPos.lng, // ✅ 添加距离、时长、费用信息 estimatedDistance: estimatedDistance, estimatedDuration: estimatedDuration, estimatedFare: estimatedFare, orderType: 'REAL_TIME' };

🔧 修复内容详解

修复1：路线规划时保存路线信息
在 planRoute() 函数中添加：
// 保存路线信息供下单使用 window.routeInfo = { distance: route.distance, // 米 duration: route.time, // 秒 price: price };
修复2：下单时包含完整信息
在 confirmOrder() 函数中添加：
// 获取路线信息（如果存在） if (window.routeInfo) { estimatedDistance = parseFloat((window.routeInfo.distance / 1000).toFixed(2)); estimatedDuration = Math.round(window.routeInfo.duration / 60); estimatedFare = parseFloat(document.getElementById('estimatedPrice').textContent.replace('¥ ', '')); }

📊 数据流对比

修复前的数据流：

1. 用户规划路线 → 显示距离、时长、费用 2. 用户点击下单 → 只发送地址和坐标 ❌ 3. 后端接收数据 → estimatedDistance = null ❌ 4. 数据库保存 → 距离、时长、费用字段为空 ❌ 5. 司机收到通知 → 没有距离和费用信息 ❌

修复后的数据流：

1. 用户规划路线 → 显示距离、时长、费用 + 保存到 window.routeInfo ✅ 2. 用户点击下单 → 发送完整的订单信息 ✅ 3. 后端接收数据 → estimatedDistance = 8.5 ✅ 4. 数据库保存 → 距离、时长、费用字段有值 ✅ 5. 司机收到通知 → 包含完整的订单信息 ✅
