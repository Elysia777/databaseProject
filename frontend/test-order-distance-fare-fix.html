🔧 订单距离费用信息修复测试

🐛 问题发现

问题描述：
乘客发起订单时可以看到距离和费用，但通知司机时没有显示，数据库也没有记录相关信息。

影响：
• 司机收到的订单通知缺少距离和费用信息
• 数据库中订单记录不完整
• 影响司机接单决策
• 后续费用计算可能不准确

📊 数据流分析

订单数据流转过程

步骤1： 乘客端计算路线信息
routeInfo.value = { distance: 8500, // 8.5公里（米） duration: 1200 // 20分钟（秒） }
步骤2： 乘客端创建订单数据
const orderData = { // ❌ 修复前：缺少距离和时长信息 estimatedFare: getPrice(selectedCarType.value) // ✅ 修复后：包含完整信息 estimatedDistance: (routeInfo.value.distance / 1000).toFixed(2), // 8.50 estimatedDuration: Math.round(routeInfo.value.duration / 60), // 20 estimatedFare: getPrice(selectedCarType.value) // 25.00 }
步骤3： 后端保存订单信息
// ✅ 修复后：保存完整的订单信息 order.setEstimatedDistance(request.getEstimatedDistance()); order.setEstimatedDuration(request.getEstimatedDuration()); order.setEstimatedFare(request.getEstimatedFare());
步骤4： WebSocket通知司机
// ✅ 通知中包含完整信息 notification.put(\"distance\", order.getEstimatedDistance()); notification.put(\"duration\", order.getEstimatedDuration()); notification.put(\"estimatedFare\", order.getEstimatedFare());

✅ 修复后的数据结构

前端订单数据：
const orderData = { passengerId: 1, pickupAddress: \"起点地址\", destinationAddress: \"终点地址\", // ✅ 添加距离信息（公里） estimatedDistance: 8.50, // ✅ 添加时长信息（分钟） estimatedDuration: 20, estimatedFare: 25.00 };
后端DTO类：
public class CreateOrderRequest { private String pickupAddress; private String destinationAddress; // ✅ 添加距离字段 private BigDecimal estimatedDistance; // ✅ 添加时长字段 private Integer estimatedDuration; // ✅ 添加费用字段 private BigDecimal estimatedFare; }

🛠️ 具体修复内容

1. 前端数据完善（PassengerMap.vue）
• 在创建订单时添加 estimatedDistance 字段
• 在创建订单时添加 estimatedDuration 字段
• 利用现有的 routeInfo 数据进行计算
2. 后端DTO类扩展
• CreateOrderRequest 添加距离、时长、费用字段
• CreateScheduledOrderRequest 添加相同字段
• 添加对应的 getter/setter 方法
3. 后端控制器修复
• OrderController 保存距离和时长信息
• ScheduledOrderController 保存距离和时长信息
• 确保数据库记录完整
4. WebSocket通知增强
• 通知中包含准确的距离信息
• 通知中包含预估时长信息
• 司机能看到完整的订单信息
