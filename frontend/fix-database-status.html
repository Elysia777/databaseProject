数据库状态字段修复指南

🚨 问题描述

创建预约单时出现错误：Data truncated for column 'status' at row 1

这是因为数据库中的 orders 表的 status 字段是 ENUM 类型，不包含 SCHEDULED 状态。

⚠️ 需要手动执行

由于安全原因，需要你手动在数据库中执行以下SQL语句来修复这个问题。

步骤 1: 连接数据库

使用你的数据库管理工具（如 MySQL Workbench、phpMyAdmin、或命令行）连接到数据库。

步骤 2: 执行修复SQL

复制并执行以下SQL语句：

-- 修复orders表的status字段，添加SCHEDULED状态 -- 使用正确的数据库（根据你的配置可能是 taxi_system 或 taxi_operation_system） USE taxi_system; -- 查看当前表结构 DESCRIBE orders; -- 修改status字段，添加SCHEDULED状态到ENUM中 ALTER TABLE orders MODIFY COLUMN status ENUM('PENDING', 'ASSIGNED', 'PICKUP', 'IN_PROGRESS', 'COMPLETED', 'CANCELLED', 'SCHEDULED') DEFAULT 'PENDING' COMMENT '订单状态：待分配、已分配、接客中、进行中、已完成、已取消、已预约'; -- 验证修改结果 DESCRIBE orders;
