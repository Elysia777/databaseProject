🔧 预约单取消司机状态修复测试

🐛 问题描述

严重Bug：
预约单司机确认到达后，如果乘客取消订单，司机会出现以下问题：
• ❌ 司机无法收到新订单（状态未释放）
• ❌ 司机一直在发送位置给已取消订单的乘客
• ❌ 实时单没有这个问题，只有预约单有问题

根本原因：
预约单取消时缺少司机状态释放逻辑，导致司机在Redis中仍然处于忙碌状态。

🔍 问题分析对比

❌ 修复前的预约单取消逻辑

// ScheduledOrderService.cancelScheduledOrder() public void cancelScheduledOrder(Long orderId, String reason) { Order order = orderMapper.selectById(orderId); Long driverId = order.getDriverId(); order.setStatus(\"CANCELLED\"); orderMapper.updateById(order); // ❌ 缺少司机状态释放 // ❌ 没有调用 driverRedisService.markDriverFree(driverId) // 只有通知，没有状态清理 webSocketNotificationService.notifyDriverOrderCancelled(driverId, orderId, reason); }

✅ 实时单的取消逻辑（正确）

// OrderController.cancelOrderByPassenger() @PostMapping(\"/{orderId}/cancel\") public Result cancelOrderByPassenger(@PathVariable Long orderId) { Order order = orderMapper.selectById(orderId); order.setStatus(\"CANCELLED\"); orderMapper.updateById(order); // ✅ 正确释放司机状态 if (order.getDriverId() != null) { driverRedisService.markDriverFree(order.getDriverId()); webSocketNotificationService.notifyDriverOrderCancelled( order.getDriverId(), orderId, \"乘客已取消订单\"); } }

🔧 修复内容详解

修复1：预约单取消时释放司机状态
在 ScheduledOrderService.cancelScheduledOrder() 中添加：
// 🔧 关键修复：释放司机状态，让司机可以接收新订单 driverRedisService.markDriverFree(driverId); log.info(\"✅ 已释放司机状态: driverId={}\", driverId);
修复2：完善markDriverFree方法
在 DriverRedisService.markDriverFree() 中添加：
// 设置司机为空闲状态 redisTemplate.opsForHash().put(DRIVER_STATUS_KEY + driverId, \"busy\", false); // 🔧 关键修复：清除司机的当前订单，停止位置推送 setDriverCurrentOrder(driverId, null);

📊 司机状态管理流程

修复后的完整流程

步骤1： 司机接单
Redis状态： - driver_status:{driverId}.busy = true - driver_current_order:{driverId} = {orderId}
步骤2： 司机确认到达（PICKUP状态）
司机开始推送位置给乘客： - isDriverBusy(driverId) = true - getDriverCurrentOrder(driverId) = {orderId} - 持续推送位置更新
步骤3： 乘客取消预约单
修复后的处理： 1. 更新订单状态为 CANCELLED 2. ✅ 调用 driverRedisService.markDriverFree(driverId) 3. ✅ 清除 driver_current_order:{driverId} 4. ✅ 设置 driver_status:{driverId}.busy = false 5. 通知司机订单已取消
步骤4： 司机状态恢复正常
结果： - 司机可以接收新订单 - ✅ 停止向乘客推送位置 - ✅ 司机重新进入空闲状态

🧪 测试验证步骤

测试场景：预约单取消后司机状态
1. 创建预约单，等待激活
2. 司机接单（状态：ASSIGNED）
3. 司机确认到达（状态：PICKUP）
4. 乘客取消预约单
5. 验证司机状态是否正确释放
关键验证点：
• 司机取消后能否立即接收新订单
• 后端日志是否停止\"推送司机位置给乘客\"
• Redis中司机状态是否正确更新
• 与实时单取消行为是否一致

🔍 调试检查点

后端日志关键词：
• \"预约单已分配给司机，释放司机状态并发送取消通知\"
• \"✅ 已释放司机状态: driverId=X\"
• \"✅ 司机 X 已标记为空闲，当前订单已清除\"
• \"已清除司机 X 当前订单\"
Redis状态检查：
• driver_status:{driverId}.busy 应该为 false
• driver_current_order:{driverId} 应该被删除
• 司机应该能查询到新的待分配订单
前端行为检查：
• 司机端应该收到订单取消通知
• 司机端应该清除当前订单状态
• 司机端应该能接收新的订单通知
